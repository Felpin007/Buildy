# AI Code Structure Generator (Text Format)

**Generate project file structures from a specific text format containing commands and code blocks, ideal for use with AI models. Paste the AI-generated text to create files and folders, with integrated Git checkpointing, undo, and auto-commit capabilities.**

This VS Code extension allows you to quickly scaffold project structures based on a structured text output generated by an AI, while also providing tools to manage the generated code.

## Features

*   **Text-Based Generation:** Parses a specific text format containing PowerShell commands (`<comando>`) and file content blocks (`<codigo ref="...">`).
*   **Dual Views:**
    *   **Structure View:** Provides an input area to paste AI-generated text and trigger the structure generation.
    *   **Copy System View:** Offers a file explorer interface within VS Code to browse your workspace, select files, and copy their contents.
*   **Git Integration:**
    *   Checks for an existing Git repository in the workspace.
    *   Prompts to initialize a Git repository (`git init`) if one doesn't exist, enabling Undo and Auto-Commit.
    *   Warns if the repository has uncommitted changes before generation.
    *   Creates a Git checkpoint commit (`AI Structure Gen: Checkpoint ...`) before generation *if* the repository is clean, allowing for a safe undo.
*   **Undo Last Generation:** Provides a command (`ai-structure-generator.undoLastGeneration`) to revert the workspace to the pre-generation checkpoint using `git reset --hard` and `git clean -fd` (use with caution!).
*   **Automatic Commits:** Automatically stages and commits file system changes (creations, modifications, deletions) detected in the workspace after a short delay, using commit messages like `Auto-commit: File system ... detected ...`.
*   **Task-Based Execution:** Executes generation steps (creating directories, writing files, running commands) via a dedicated VS Code Task using PowerShell, showing colored status output in the terminal.
*   **Format Handling:** Ignores informational tags like `<texto>` and `<estrutura>` during processing.

## How to Use

1.  **Generate the Structure Text:** Use an AI model capable of generating code and following specific formatting instructions (e.g., Google Gemini). Provide the AI with the following prompt structure. **Note:** While the prompt includes instructions for editing, the extension currently focuses primarily on the **creation** aspect described in "PARA CRIAÇÃO".

    ```text
    Responda no seguinte formato seguindo as instruções abaixo:

    PARA CRIAÇÃO:
    ------------------------------------------------------------------
    A estrutura de paginas (não crie pasta raiz, ex: projeto-carros):

    PARTE ESTRUTURAL: <- obrigatóriamente como primeiro
    <texto>aqui está a estrutura:</texto>
    (estrutura em ASCII)
    ex:
    <estrutura>
    ...estrutura...
    </estrutura>

    <comando>mkdir...</comando> <- pode ter comandos em qualquer lugar

    <codigo ref="./index.html"> <- essa tag cria o arquivo, não use touch.
    <html>
    ....
    </html>
    </codigo>

    <codigo ref="./monstro.svg">
    <svg>
    ...
    </svg>
    </codigo>

    <comando>...</comando>
    <comando>...</comando>

    <codigo ref="./index.html">
    <html>
    ....
    </html>
    </codigo>

    <comando>...</comando>

    Mostre SEMPRE o código COMPLETO.

    -------------------------------------------------------------

    PARA EDIÇÃO + adição (opcional):
    -------------------------------------------------------------
    Responda com:
    ESTRUTURA atualizada (obs: todo arquivo novo/existente que sofreu alteração deve ser mostrado COMPLETO/ Arquivos sem alteração não devem serem mostrados)
    obs: ao mostrar a estrutura de edição aponte o que manteve igual e o que sofreu alteração, ex: tal_arquivo.slaoq <- mantem igual, etc...

    Justificativa para edições e mudanças<texto>Vamos alterar tal arquivo por que ... </texto>

    RESTANTE SEGUINDO AS MESMAS REGRAS ANTERIORES

    obs geral:qualquer coisa que você for falar de texto normal para o usuario, fale sob a tag <texto></texto>
    obs geral 2: nunca abrevie o código ou pule alguma parte.
    ```

2.  **Copy the Text:** Copy the entire relevant text output generated by the AI, typically starting from the `PARTE ESTRUTURAL:` line.

3.  **Open the Structure View:** In VS Code, click on the "AI Structure Gen" icon in the Activity Bar, then select the "Structure" view.

4.  **Prepare Git (if applicable):**
    *   If your workspace folder isn't a Git repository, you'll be prompted to initialize one when you generate. Choosing "Inicializar Git" is recommended for Undo and Auto-Commit features.
    *   If your repository has uncommitted changes, you'll be warned. Auto-commit will include these changes. You can choose to "Continuar" (disabling safe Undo) or "Cancelar Geração" to commit/stash manually first.

5.  **Paste and Generate:** Paste the copied text into the text area within the "Structure" view and click the "Generate Structure" button.

6.  **Monitor the Task Terminal:** A terminal named "AI Structure Generation" will open (or be reused) and display colored output showing the progress of directory creation, file writing, and command execution relative to your workspace folder. Check this terminal for any errors.

7.  **Use Undo (if needed):** If the generation fails, or you want to revert the changes, and a clean checkpoint was created (step 4), you can run the command `AI Structure Gen: Undo Last Generation` from the Command Palette (Ctrl+Shift+P). **Warning:** This performs a hard reset and cleans untracked files, potentially causing data loss if used incorrectly.

8.  **Use Copy System View:** Select the "Copy System" view in the "AI Structure Gen" container to browse your workspace files. Checkboxes allow you to select files, and the "Copy Selected Files" button ( <i class="codicon codicon-clippy"></i> ) copies the content of the selected files to your clipboard, formatted with file paths.

9.  **Auto-Commit:** After the initial generation, subsequent changes you make to files (saving, creating, deleting) within the workspace will be automatically staged and committed by the extension after a short delay.

    *Example of expected AI output format (similar to before, aligning with the new prompt's structure):*
    ```text
    PARA CRIAÇÃO:
    ------------------------------------------------------------------
    PARTE ESTRUTURAL:
    <texto>Aqui está a estrutura básica para o projeto web:</texto>
    <estrutura>
    .
    ├── css/
    │   └── style.css
    ├── js/
    │   └── script.js
    └── index.html
    </estrutura>

    <comando>mkdir css</comando>
    <comando>mkdir js</comando>

    <codigo ref="./index.html">
    <!DOCTYPE html>
    <html lang="pt-BR">
    <head>
        <meta charset="UTF-8">
        <title>Meu Projeto</title>
        <link rel="stylesheet" href="css/style.css">
    </head>
    <body>
        <h1>Olá, Mundo!</h1>
        <script src="js/script.js"></script>
    </body>
    </html>
    </codigo>

    <codigo ref="./css/style.css">
    body {
        font-family: sans-serif;
        margin: 20px;
    }
    h1 {
        color: navy;
    }
    </codigo>

    <codigo ref="./js/script.js">
    console.log("Página carregada!");
    // Código JavaScript adicional aqui
    </codigo>

    <comando>Write-Host "Estrutura básica criada com sucesso!" -ForegroundColor Green</comando>
    ```

## Important Notes

*   **PowerShell Required:** Ensure you have **PowerShell** available on your system and accessible in your VS Code terminal environment.
*   **Format Sensitivity:** The extension strictly relies on the `<comando>...</comando>` and `<codigo ref="...">...</codigo>` tags. Malformed tags or incorrect `ref` attributes will likely cause errors.
*   **Ignored Tags:** Content within `<texto>...</texto>` and `<estrutura>...</estrutura>` is ignored by the generation process.
*   **Execution Order:** Operations are executed **sequentially** as they appear in the pasted text. Ensure logical order (e.g., `mkdir` before writing files inside).
*   **Workspace Context:** All operations are relative to the root of the currently opened workspace folder.
*   **Overwriting Files:** `<codigo>` blocks referring to existing files **will overwrite** their content (`Set-Content -Force`).
*   **Security:** Executing commands (`<comando>`) generated by an AI carries risks. **Always review the commands** before generating to prevent unintended actions.
*   **Git Integration:**
    *   Initialization (`git init`) is offered if no repo exists.
    *   Checkpoints are only created if the repository is clean before generation.
    *   Auto-commit applies to all changes detected after a delay. Commit messages are generic (`Auto-commit: ...`).
*   **Undo Functionality:**
    *   Requires a clean Git checkpoint created by the extension before generation.
    *   Uses `git reset --hard <checkpoint>` and `git clean -fd`.
    *   **Warning:** This permanently deletes uncommitted changes in tracked files and *all* untracked files/directories. Use with extreme caution.
*   **Copy System:** Copies the *content* of selected files, useful for feeding back into AI prompts or documentation.
